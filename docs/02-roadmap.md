# Country Explorer — Роадмап

## Принцип приоритизации

Роадмап построен по принципу **«от ценности к полноте»**: каждая итерация должна приносить пользу клиенту, а не просто «доделывать» предыдущую. Приоритизация через RICE-подобный подход: Reach (сколько клиентов используют) × Impact (насколько меняет поведение) × Confidence / Effort.

---

## Phase 1: MVP — «Покажи мне рынок» (4 недели)

**Цель:** Клиент впервые видит полную картину присутствия брендов в стране.

**Что делаем:**
1. Хороплетная карта регионов (ITL1) с подсветкой по количеству/плотности
2. Панель выбора брендов (toggle on/off)
3. Три метрики для окраски: абсолютное количество, на 100к населения, доля бренда
4. Region drill-down: клик → панель с breakdown по брендам, доля рынка, doughnut chart
5. Табличный вид с сортировкой по любой колонке
6. Отображение точек на карте (переключение regions/points/both)

**Почему это первое:**
- Покрывает JTBD #1 и #2 (обзор рынка + сравнение с конкурентами)
- Минимальный набор данных (у нас уже есть всё необходимое)
- Быстрый time-to-value для клиента
- Позволяет собрать обратную связь до инвестиций в сложные фичи

**Метрики успеха:** WAU > 60% от пользователей с доступом, avg session > 3 мин, >2 региона просмотрено за сессию

---

## Phase 2: Глубина — «Покажи детали» (3 недели)

**Цель:** Клиент может копать глубже — до городов и отдельных точек.

**Что делаем:**
1. Drill-down до уровня города (кластеризация точек по городам)
2. Popup точки: адрес, бренд, атрибуты (drive-thru, delivery, рейтинг)
3. Фильтрация по атрибутам (только с delivery, только с drive-thru)
4. Экспорт данных: CSV таблицы, PDF snapshot текущего вида
5. Population overlay: нормализация по населению городов/регионов
6. Поиск по названию бренда, города, postcode

**Почему второе:**
- Expansion managers не могут принимать решения на уровне регионов — нужны города
- Фильтры по атрибутам — ключевой запрос от клиентов (из discovery)
- Экспорт — необходимость для презентаций руководству

**Метрики успеха:** export rate > 20%, drill-down depth avg > 2 уровня

---

## Phase 3: Динамика — «Что изменилось?» (4 недели)

**Цель:** Клиент видит не снимок, а фильм — тренды, открытия, закрытия.

**Что делаем:**
1. Timeline slider — выбор периода, анимация изменений на карте
2. «Changes» tab — список открытий и закрытий за период
3. Trend charts по регионам (рост/падение кол-ва точек)
4. Email alerts — уведомления при открытии/закрытии конкурентов в выбранных регионах
5. Comparison mode — два периода рядом (before/after)

**Почему третье:**
- Требует исторических данных (нужна pipeline для сбора)
- Alerts — sticky feature, повышающая retention
- Динамика превращает инструмент из «посмотрел раз» в «открываю каждую неделю»

**Метрики успеха:** alert setup rate > 30%, return visit within 7 days > 70%

---

## Phase 4: Масштаб — «Покажи мне весь мир» (6 недель)

**Цель:** Мультистрановой анализ, кастомизация, интеграции.

**Что делаем:**
1. Добавление новых стран (DE, FR, ES, IT, AE, SA)
2. Cross-country comparison (один бренд в разных странах)
3. Custom brand groups (например, «мои конкуренты в пицце»)
4. API для выгрузки данных в BI-системы клиента
5. White-label возможность (кастом брендинг для enterprise)

**Почему четвёртое:**
- Мультистрановость — major expansion, требует отдельной data pipeline
- API — длинный цикл продаж, но высокий LTV
- White-label — enterprise upsell

## Риски и митигации

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Тяжёлые GeoJSON тормозят карту | Высокая | TopoJSON + vector tiles + серверная кластеризация |
| Нет исторических данных для Phase 3 | Средняя | Начать сбор snapshots с Phase 1, пока копим историю |
| Клиенты хотят города раньше регионов | Средняя | Быстрый follow-up Phase 2, готовить city-level данные параллельно с Phase 1 |
| Разные форматы данных по странам | Высокая | Универсальная схема нормализации с Phase 1 |
